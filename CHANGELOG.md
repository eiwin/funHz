## Git 提交规范
提交信息格式：
```
<type>(<scope>): <subject>

<body>
```

类型(type)：
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 代码格式调整
- refactor: 代码重构
- test: 测试相关
- chore: 构建过程或辅助工具的变动

示例：
```
feat(data): 添加基础汉字数据
fix(voice): 修复iOS语音播放问题
docs(changelog): 更新项目进展日志
```
